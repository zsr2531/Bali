Enum JvmOperandType

An enumeration representing the type of the operand of a JvmOpCode.

Namespace: Bali.Emit
Assembly: Bali.dll
Syntax
public enum JvmOperandType : byte

Fields

Name Description
ArrayType

Indicates that the opcode's operand is a JvmPrimitiveArrayType.
BranchOffset

Indicates that the opcode has a signed short operand that is the offset to the target instruction, relative to the current one.
ConstantPoolIndex

Indicates that the opcode's operand indexes the ConstantPool with a single unsigned byte as the index.
IndexJumpTable

Indicates that the opcode has a jump table operand.
InlineByte

Indicates that the opcode's operand is a signed byte.
InlineShort

Indicates that the opcode's operand is a signed short.
KeyJumpTable

Indicates that the opcode has a jump table operand.
LocalIndex

Indicates that the opcode's operand references an argument/local variable with a single unsigned byte as the index.
LocalIndexWithSignedByte

Indicates that the opcode's operand references an argument/local variable with an unsigned short as the index, but also has an extra signed byte.
None

Indicates that the opcode has no operand.
Undefined

Indicates that the opcode's operand is not defined.
WideBranchOffset

Indicates that the opcode has a signed int operand that is the offset to the target instruction, relative to the current one.
WideConstantPoolIndex

Indicates that the opcode's operand indexes the ConstantPool with an unsigned short as the index.
WideConstantPoolIndexWithArrayDimensions

Indicates that the opcode's operand indexes the ConstantPool with an unsigned short as the index, but also has an extra unsigned byte for the array dimensions.
WideConstantPoolIndexWithTwoBytes

Indicates that the opcode's operand indexes the ConstantPool with an unsigned short as the index, but also has 2 extra unsigned bytes.
